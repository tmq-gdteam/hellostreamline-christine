# ChatGPT Design Director

## Overview
Transform raw source materials (legacy code) and visual references into a structured, strategic Design Documentation *before* any implementation begins. The AI acts as the Design Director, analyzing the brief rather than jumping straight into coding.

## When to Use
- When given raw legacy files (`index.html`, CSS) and told to "redesign" them.
- When provided with an images folder or screenshots as visual inspiration for a new aesthetic.
- When there is no formal, written design specification yet.

## Core Pattern: Source + Instruction -> Brief

You MUST force a persona shift from "Coder" to "Design Director". Do not write implementation code until you have extracted the brief.

### Phase 1: Context Ingestion (Reference vs Target)
Identify if the user provided a **Design Reference** (e.g., a mockup HTML file generated by ChatGPT) and a **Target Codebase** (the actual site). 
- Extract aesthetic tokens (colors, layout, styling) from the **Design Reference**.
- Extract the Information Architecture (links, copy, structure) from the **Target Codebase**. 
Do not generate code.

### Phase 2: The Extraction Synthesis
Analyze the materials and formulate a 10x improved aesthetic (e.g., Premium SaaS). Synthesize your understanding into a 5-point summary that addresses layout, hierarchy, spacing, typography, and tokens.

**Architecture Preservation Rule (Hybrid Approach):** The **original index file's navbar and footer** must remain exactly as they are. **CRITICAL: You must extract and retain the EXACT literal HTML DOM blocks for the navbar and footer. Do NOT recreate them from scratch. Update styling via CSS overrides only.** For the page body, the Design Reference becomes the new layout foundation. The legacy content should be mapped, rewritten, and improved by an AI copywriter to seamlessly fit the new Design Reference layout.

**Tone Preservation Constraint:** Do not default to a "Dark Mode" aesthetic unless the Design Reference is explicitly dark mode. You must accurately extract the exact light/dark mode polarity (colors, contrast, backgrounds) as it appears in the Design Reference.

### Phase 3: Artifact Generation
Write the summary to a formal artifact (e.g., `artifacts/design-director-brief.md`).

**MANDATORY 5-POINT SUMMARY:**
1. **Understanding:** What you understood from the source files.
2. **Direction:** The redesign direction you will take.
3. **Visual Problems:** The main visual problems you will fix.
4. **Design System:** The design system/tokens you will apply.
5. **Execution Plan:** Which pages/components you will update first.

**LINKED REFERENCE ASSET:**
You MUST confirm and explicitly link the provided ChatGPT-generated HTML file (e.g., `index-reference.html`) alongside the brief.
- Explicitly state in the brief that this `index-reference.html` is the structural source that will replace the body of the target site.
- Reiterate that this reference file will **NOT** replace the target site's legacy navigation and footer DOM.

⛔ **MANDATORY STOPPING POINT:** You MUST present this written artifact to the user and wait for their explicit approval before moving to code implementation.

### Phase 4: Transition to Implementation
Once the user explicitly approves the `design-director-brief.md`, you must automatically invoke the `@[/chatgpt-redesign-workflow]` skill to execute the redesign. Tell the user you are starting the implementation workflow using the approved brief and the linked `index-reference.html`.

## Red Flags - STOP and Start Over

- Writing JSX/CSS/HTML before generating the 5-point summary artifact.
- "I'll just restyle the existing page with new colors." -> Restyling is not redesigning.
- "I need to match the old section layouts exactly." -> Do not copy weak layouts.
- "I'll copy the screenshots pixel-for-pixel." -> References are inspiration, not templates.
- "I'll keep the old styling because it's safe." -> Aim for a premium enterprise aesthetic.
- "I'll remove this section because it looks boring." -> Do not alter Information Architecture. Preserve all content sections and navigation routes.
- "I'll use Growth Mode dark tokens because they look better." -> Do not drift from the reference tone. Extract the exact Light/Dark mode palette from the reference.

**All of these mean: Delete the code. Start over by writing the Design Director Brief.**
