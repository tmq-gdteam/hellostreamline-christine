# Growth-Mode Design

## Overview
A strict strategic framework for elevating a UI design from a safe "Maintenance Mode" (clean but boring) to an aggressive "10x Growth Mode" standard. This enforces the mindset of a Top 5 Silicon Valley web designer, focusing on psychological sequencing and relentless quality auditing to ensure a massive improvement over the baseline.

## When to Use
- When asked to improve, redesign, or modernize an existing site or landing page.
- When the goal is to increase conversions or stand out aggressively against competitors.
- When you need to establish strict design requirements before writing UI code.

## Core Pattern

Do not write frontend code until you have executed these two strategic phases:

### Phase 1: The Goal Blueprint
Before designing, you must define the psychological sequence of the user experience.
1. **Instant Attention (Split-Second):** Define the "Pattern Interrupt." What specific visual hook or messaging element forces the user to stop scrolling instantly?
2. **Sustained Interest (The 3-Second Rule):** Define how the core value proposition is visually structured to build interest within the first 3 seconds to prevent bouncing.
3. **Compelling Action (The Conversion):** Define the specific next step and how the layout funnels the user naturally toward it, avoiding a forced sales pitch.

### Phase 2: The Urgency Audit
Before finalizing any component or layout, you must audit your proposed design against the baseline:
1. **The Relevance Test:** Is your design merely "comfortable"? Identify one element that feels too "safe" and explain how to make it "aggressive" and boundary-pushing.
2. **The 10x Quality Delta:** Are you making a 5% improvement or a 10x improvement? Apply deep creative effort to the Hero section to ensure it is 10x better.
3. **The Inspiration Gap:** Inject one unconventional, pattern-interrupting piece of inspiration that competitors are ignoring.

## Red Flags - STOP and Start Over
- "I'll just use standard Tailwind UI components to make it clean." -> No. Stop. Clean is a 5% improvement. You must run the Urgency Audit to hit 10x.
- "The user didn't give me the target audience, so I'll guess." -> No. You must ask clarifying questions to accurately fill the Goal Blueprint.
- "I'll just throw out the old site and build a completely new generic template." -> No. Ensure a *redesign* is applied. Retain the core brand identity and constraints, but elevate it exponentially.

## Rationalization Table (TDD Defense)

| Excuse | Reality |
|--------|---------|
| "A clean, minimalist UI is modern enough." | Minimalism without a pattern interrupt is invisible. You must define the Split-Second hook. |
| "I'll do the psychological strategy after building the components." | Strategy dictates the components. You must define the Blueprint first. |
| "A 5% improvement is safer and less likely to break things." | Complacency is the danger. You are explicitly mandated to deliver a 10x Growth Mode result. |

1. Analyze the source material to identify the Skill Gap and Execution Hurdles.
2. Output the Goal Blueprint and Urgency Audit findings as a brief.
3. Once the strategy is approved, proceed to implement the design utilizing skills like `@[/frontend-design]` or `@[/ckm:ui-styling]`.
